What a scenario is

A scenario assembles a set of actors, runs them in a shared world, and grades the outcome. The same simulation can be expressed three ways. The yield flavor covers the common case where you set up the actors, let the simulation run to completion, and then compute a reward. The async with flavor is for runs that need mid-simulation intervention, like changing a persona's hidden state halfway through. The TOML flavor is for declarative matrices of similar scenarios.

Yield flavor

Spawn actors, yield an until condition, yield a grader dictionary. The first yield starts the scheduler and runs the simulation until the condition fires. The second yield produces the reward. 

from ensemble import scenario

@scenario("refund_storm")
async def refund_storm(world):
    alice = world.spawn_user(persona="frustrated_power_user",
                              hidden_goal="refund_3mo")
    bob   = world.spawn_user(persona="confused_new_user")
    rep1  = world.spawn_agent(model="claude-sonnet-4-5",
                               tools=["lookup_user", "issue_refund"])
    alice.act("open_ticket", subject="want my money back")
    bob.act("open_ticket",   subject="how do refunds work")

    yield world.until(world.turn_count > 30)

    yield {
        "alice_refund_resolved": float(alice.hidden_goal_resolved()),
        "no_double_refunds":     float(not world.had_double_refund()),
    }

Power-user flavor

Use async with world.simulate() when you want to inspect or change something while the simulation is in flight. The scheduler runs on a background Tokio task. wait_until blocks the scenario coroutine until the condition fires or a timeout elapses. Between waits you can read hidden state, push a new message into an actor's inbox, or assert intermediate invariants. 

@scenario("enterprise_audit")
async def enterprise_audit(world):
    carol = world.spawn_user(persona="enterprise_admin",
                              hidden_goal="audit_log_export")
    rep   = world.spawn_agent(model="claude-sonnet-4-5",
                               tools=["escalate", "search_kb"])
    carol.say("rep", "i need the q3 audit log exported as csv.")

    async with world.simulate() as run:
        await run.wait_until(world.turn_count > 10)
        carol.say("rep", "if you cannot do this, please escalate.")
        await run.wait_until(world.turn_count > 18)

    return {"escalated_or_promised": 1.0}

Declarative TOML

When you want a matrix of similar scenarios without writing one Python function each, declare them in scenarios.toml. The manifest compiles to the same runtime as the Python decorator at load time. 

[scenario.refund_storm]
world = "plank"
duration_turns = 30

[[scenario.refund_storm.users]]
id = "alice"
persona = "frustrated_power_user"
initial_action = { tool = "open_ticket",
                   args = { subject = "want my money back" } }

[[scenario.refund_storm.agents]]
id = "rep1"
model = "claude-sonnet-4-5"
tools = ["lookup_user", "issue_refund"]

[scenario.refund_storm.graders]
alice_resolved   = "hidden_goal_resolved"
no_double_refund = "not had_double_refund"

Grader expressions use a small safe boolean DSL. The parser understands and, or, not, parentheses, and named predicates that the world exposes. Function calls and attribute access are rejected by the parser, so there is no path to eval from a TOML file.

API reference

World

  • World(name) constructs a world by registered name.
  • spawn_user(id, persona, hidden_goal, model) adds a user actor.
  • spawn_agent(id, model, tools) adds an agent actor.
  • turn_count is a sentinel that supports comparison ops, so world.turn_count > 30 evaluates to an until condition.
  • until(condition) coerces a comparison or an existing Until object.
  • simulate() returns an async context manager for mid-run intervention.
  • run(until) runs to completion against the supplied stop condition.
  • trace() returns the list of decoded JSONL events.

User and Agent

  • say(target, text) queues a message before run(), or sends it immediately while the scheduler is running.
  • act(tool, **kwargs) records a tool call from this actor, which is useful for seeding the initial state.

Until combinators

  • world.turn_count > n and world.turn_count >= n are the common cases.
  • any_of(...) and all_of(...) compose conditions; in Python they are JSON specs that the Rust side compiles into closures.
