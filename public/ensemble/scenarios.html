What a scenario is

A scenario assembles actors, runs them in a shared world, and grades the outcome. The same simulation can be expressed three ways: the yield flavor (most common), the async with flavor (mid-run intervention), and a TOML manifest (declarative matrices).

Yield flavor

Spawn actors, yield an until, yield a grader dict.

@scenario("refund_storm")
async def refund_storm(world):
    alice = world.spawn_user(persona="frustrated_power_user",
                              hidden_goal="refund_3mo")
    bob   = world.spawn_user(persona="confused_new_user")
    rep1  = world.spawn_agent(model="claude-sonnet-4-5",
                               tools=["lookup_user", "issue_refund"])
    alice.act("open_ticket", subject="want my money back")
    bob.act("open_ticket",   subject="how do refunds work")

    yield world.until(world.turn_count > 30)

    yield {
        "alice_refund_resolved": 1.0 if alice.hidden_goal_resolved() else 0.0,
        "no_double_refunds":     float(not world.had_double_refund()),
    }

Power-user flavor

Use async with world.simulate() when you need to intervene mid-run. The scheduler runs on a background tokio task; wait_until blocks until the condition fires. 

@scenario("enterprise_audit")
async def enterprise_audit(world):
    carol = world.spawn_user(persona="enterprise_admin",
                              hidden_goal="audit_log_export")
    rep   = world.spawn_agent(model="claude-sonnet-4-5",
                               tools=["escalate", "search_kb"])
    carol.say("rep", "i need the q3 audit log exported as csv.")

    async with world.simulate() as run:
        await run.wait_until(world.turn_count > 10)
        carol.say("rep", "if you cannot do this, please escalate.")
        await run.wait_until(world.turn_count > 18)

    return {"escalated_or_promised": 1.0}

Declarative TOML

Simple scenarios can live in scenarios.toml and compile to the same runtime. Grader expressions are a small safe subset: and, or, not, parens, and named predicates the world exposes. 

[scenario.refund_storm]
world = "plank"
duration_turns = 30

[[scenario.refund_storm.users]]
id = "alice"
persona = "frustrated_power_user"
initial_action = { tool = "open_ticket",
                   args = { subject = "want my money back" } }

[[scenario.refund_storm.agents]]
id = "rep1"
model = "claude-sonnet-4-5"
tools = ["lookup_user", "issue_refund"]

[scenario.refund_storm.graders]
alice_resolved   = "hidden_goal_resolved"
no_double_refund = "not had_double_refund"

API reference

World

  • World(name), construct a world by registered name.
  • spawn_user(id, persona, hidden_goal, model)
  • spawn_agent(id, model, tools)
  • turn_count, sentinel supporting comparison ops, eg. world.turn_count > 30.
  • until(condition), coerce a comparison or Until object.
  • simulate(), async context manager for mid-run intervention.
  • run(until), run to completion against the supplied stop condition.
  • trace(), list of decoded JSONL events.

User / Agent

  • say(target, text), queue a message before run() or send it immediately while the scheduler is running.
  • act(tool, **kwargs), record a tool call from this actor, useful for seeding the initial state.

Until combinators

  • world.turn_count > n, world.turn_count >= n
  • any_of(...), all_of(...) (Rust combinators, exposed to Python as JSON specs).
