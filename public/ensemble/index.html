ensemble

A framework for building RL environments where several simulated users and several agents share one typed world. You describe the world in Rust, you write the scenarios in Python, and the framework handles message routing, tool dispatch, the trace log, and the grader that turns each rollout into a reward.

Why it exists

Most agent benchmarks model one agent talking to one user in a stateless sandbox. Production support, sales, customer success, and operations all look very different. There are multiple users contending for attention, multiple agents seeing each other's tool calls, shared state that changes underfoot, and policies that only become visible across many turns. Ensemble lets you build that, run rollouts at scale, and turn the rollouts into training signal.

Install

uv sync
cargo build -p ensemble-cli

The docs page has the full quickstart, with a walkthrough of how to write a world, a scenario, and a grader, and how to use the framework as an RL environment.
