What Plank is

Plank is a small fake SaaS customer-support tool and the worked example world for Ensemble. It exists to demonstrate the framework's shape end to end: a typed world, a tool registry, a handful of personas, and three scenarios that stress different parts of the system.

State

An in-memory SQLite database with six tables:

users         (id, name, email, plan)
tickets       (id, user_id, subject, status, opened_at)
subscriptions (user_id, plan)
kb            (id, title, body)
audit_log     (id, ts, who, action, payload)
refunds       (id, user_id, amount_cents, reason, ts)

The state lives inside plank::PlankState, behind an Arc<Mutex<_>> shared across all tool closures. Each World("plank") instance gets its own fresh state.

Tools

  • lookup_user(user_id), fetch a user record.
  • lookup_ticket(ticket_id), fetch a ticket.
  • issue_refund(user_id, amount_cents, reason), one per user per run; the second call errors.
  • escalate(ticket_id, to_team), sets status to escalated.
  • search_kb(query), case-insensitive substring search.
  • update_subscription(user_id, plan), switch a user's plan.

All six audit every call into audit_log. The no_double_refunds grader on the front-page demo reads back from refunds to verify the policy held.

Personas shipped

  • frustrated_power_user, trained adapter via DPO+LoRA.
  • confused_new_user, prompted.
  • enterprise_admin, prompted.
  • patient_retail, prompted.
  • chatty_neutral, prompted.

See the personas page for the TOML schema and how training works.

Scenarios shipped

  • plank.single_ticket, one user, one agent. Sanity check.
  • plank.refund_storm, three users, two agents. The hero demo on the front page.
  • plank.enterprise_audit, async with path with mid-run intervention.

Run it

cargo build -p ensemble-cli
./target/debug/ensemble run plank.refund_storm --world plank
./target/debug/ensemble trace view traces/plank_refund_storm.jsonl \
  --site site --port 8765

Source: examples/plank/. The Rust world is at examples/plank/world/src/; the Python scenarios are at examples/plank/scenarios/; the personas are TOML files under examples/plank/personas/.
