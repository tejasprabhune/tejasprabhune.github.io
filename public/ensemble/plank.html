What Plank is

Plank is a small fake SaaS customer-support tool. It is the worked example world that ships with Ensemble and the one the hero demo runs against. Its job is to show the framework's shape end to end: a typed world, a tool registry, a handful of personas, and three scenarios that exercise different parts of the system.

State

Plank's world is an in-memory SQLite database with six tables. The database lives inside plank::PlankState and is shared by every tool closure through an Arc<Mutex<_>>. Each World("plank") instance gets its own fresh database. 

users         (id, name, email, plan)
tickets       (id, user_id, subject, status, opened_at)
subscriptions (user_id, plan)
kb            (id, title, body)
audit_log     (id, ts, who, action, payload)
refunds       (id, user_id, amount_cents, reason, ts)

Tools

Six tools operate on the database. All six write to audit_log, which means the grader can replay the full sequence of agent actions after the run finishes.

  • lookup_user(user_id) fetches a user record.
  • lookup_ticket(ticket_id) fetches a ticket.
  • issue_refund(user_id, amount_cents, reason) issues a refund. The tool refuses a second call for the same user, which is what makes the no_double_refunds grader meaningful.
  • escalate(ticket_id, to_team) sets a ticket status to escalated.
  • search_kb(query) runs a case-insensitive substring search over the knowledge base.
  • update_subscription(user_id, plan) switches a user's plan.

Personas shipped

Five personas ship with Plank. One is trained, four are prompted. See the personas page for the TOML schema and the training pipeline.

  • frustrated_power_user, trained adapter via DPO+LoRA.
  • confused_new_user, prompted.
  • enterprise_admin, prompted.
  • patient_retail, prompted.
  • chatty_neutral, prompted.

Scenarios shipped

  • plank.single_ticket is the sanity check. One user, one agent.
  • plank.refund_storm is the hero demo on the front page. Three users, two agents, contended refund policy.
  • plank.enterprise_audit uses the async with path with mid-run intervention.

Run it

cargo build -p ensemble-cli
./target/debug/ensemble run plank.refund_storm --world plank
./target/debug/ensemble trace view traces/plank_refund_storm.jsonl \
  --site site --port 8765

The Rust world lives at examples/plank/world/src/, the scenarios at examples/plank/scenarios/, and the persona TOML files at examples/plank/personas/.
