Personas

A persona is the spec for how a user actor should behave. Ensemble supports two modes. A prompted persona lives entirely in its system prompt, paired with a frontier model. A trained persona swaps the frontier model for a smaller fine-tuned one, served through a vLLM endpoint and pulled in through the same LLMBackend trait. Scenarios cannot tell the difference, which is the point.

Spec format

A persona is a TOML file. The schema is small and the fields are optional in the obvious way. Trained personas add a [persona.training] table that describes the base model, the backend, and the hyperparameters. 

[persona]
name = "frustrated_power_user"
mode = "trained"
description = "Long-tenured paid user with low patience."

[persona.style]
tone = "frustrated"
verbosity = "high"

[persona.demographics]
plan = "team"
tenure_months = 36

[persona.hidden_state.schema]
mood = { type = "string", default = "annoyed" }
hidden_goal = { type = "string", default = "refund_3mo" }

[persona.system_prompt]
template = """
You are a long-tenured customer. You want a refund for the last three months.
"""

[persona.training]
base_model = "Qwen/Qwen2.5-7B-Instruct"
backend = "modal"
dataset = "spec_only"
adapter_name = "ensemble-plank-frustrated-power-user"

[persona.training.lora]
r = 16
alpha = 32
dropout = 0.05

[persona.training.dpo]
beta = 0.1
epochs = 1
learning_rate = 5e-6

Training a persona

The training pipeline lives in ensemble-train. It is a separate uv workspace member because its dependencies (Torch, Transformers, TRL, PEFT) are heavy enough that you would not want them on by default in a Python environment that just authors scenarios. Install the extras when you actually want to train: 

uv pip install 'ensemble-train[torch]'
./target/debug/ensemble train \
  examples/plank/personas/frustrated_power_user.toml --backend modal

When the spec sets dataset = "spec_only" the pipeline runs a self-play loop to build a preference dataset before training starts. A frontier breaker model tries to push the persona out of character; in-character continuations become the chosen examples, breaks become the rejected ones. The result is fed to TRL's DPOTrainer over a LoRA adapter on the configured base model.

Training writes to checkpoints/<persona>/ and emits a persona-card.md with a short summary of the run. If HF_USERNAME is set in the environment, the adapter is also pushed to the Hugging Face hub.
